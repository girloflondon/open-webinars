Создаем HTML-разметку

Создаем файл

Открываем текстовый редактор и создаем пустой документ. Сохраняем его в нужную папку под именем index.html.

Пишем обязательную разметку

Все HTML-страницы должны содержать несколько обязательных строк разметки. Напишем их в пустом файле.

        
          <!DOCTYPE html>
          <html lang="ru">
          <head>
            <meta charset="UTF-8">
            <title>Члены семьи Симпсонов</title>
          </head>
          <body>
          </body>
          </html>

Если используете Emmet: напишите ! затем нажмите Tab.

Виды DOCTYPE

DOCTYPE, а точнее его атрибут указывает браузеру как следует интерпретировать HTML-документ.

<!DOCTYPE html> HTML5, применяется для всех документов
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd"> Строгий синтаксис HTML
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd"> Переходный синтаксис HTML
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd"> Строгий синтаксис XHTML

main

Создаем основной родительский блок при помощи тега main:

        
          <main>
          </main>

main – тег, который включает в себя содержимое, являющееся уникальным для данного документа, и не должен включать повторяющиеся разделы сайта, такие как навигация, название сайта, логотип, поисковая форма, баннеры и др.

section

Внутрь родительского блока main вложим несколь секций section. Сколько секций будет в разметке, на столько частей и будет разделен баннер.

        
          <main>
            <section>
            </section>
            <section>
            </section>
            <section>
            </section>
            <section>
            </section>
            <section>
            </section>
          </main>

section

section – задаёт раздел документа, может применяться для блока новостей, контактной информации, глав текста, вкладок в диалоговом окне и др. Раздел обычно содержит заголовок.

Блоки для текста

На каждой секции баннера будет заголовок, подзаголовок и описание. Создадим для этого по два блока в каждой секции, зададим им соответствующие классы:

        
          <section>
            <div class="cont_title">
            </div>
            <div class="cont_desc">
            </div>
          </section>

Блоки для текста

Класс в CSS – задается при помощи атрибута class для любого HTML-тега и позволяет связать этот тег со свойствами CSS-стилей. Помогает различать одинаковые теги и присваивать им различные стили.

div – является универсальным блочным элементом и предназначен для группирования элементов документа с целью изменения вида содержимого через стили.

Текстовые теги

Теперь добавим в блок с классом .cont_title два тега для заголовка (h2) и подзаголовка (h3).

В блок с классом .cont_desc добавим абзац (p).

        
          <section>
            <div class="cont_title">
              <h2>Гомер</h2>
              <h3>Глава семьи</h3>
            </div>
            <div class="cont_desc">
              <p>Работает на Спрингфилдской атомной электростанции.
               Больше всего любит пиво, пончики и жену Мардж.</p>
            </div>
          </section>

Текстовые теги

h1-h6 – заголовки предназначены для использования вместе с абзацами и списками, чтобы нарисовать естественную структуру вашего документа. Поскольку заголовки несут больший семантический вес, будьте осторожны, сохраняя баланс между важным и «обычным» содержанием.

p – определяет текстовый абзац. Элемент p является блочным, всегда начинается с новой строки, абзацы текста идущие друг за другом разделяются между собой отбивкой.

Создаем и подключаем файл

В текстовом редакторе создаем файл, сохраняем его в ту же папку, что и index.html под названием style.css.

В файле index.html в шапке сайта (между <head> и </head>) подключаем файл стилей:

        
          <head>
            <meta charset="UTF-8">
            <title>Члены семьи Симпсонов</title>
            <link rel="stylesheet" href="style.css">
          </head>

Абсолютные пути

Абсолютный путь – полный путь до конкретного файла на сервере. Должен начинаться с указания протокола (http://). Вы идете в незнакомое место, но у вас есть точный адрес.

Пример: <a href="http://netology.ru/">Нетология</a>

Относительные пути

Относительный путь – указывает расположение файла относительно текущей позиции. Чтобы из дома дойти до булочной нужно пройти два дома и повернуть направо. Если вы находитесь не дома, то подобные указания уведут вас в другое место.

Примеры:

  • Файлы находятся в одной папке:
    <a href="style.css">Стили</a>
  • Файл находится выше уровнем:
    <a href="../style.css">Стили</a>
  • Файл находится во вложенной папке:
    <a href="css/style.css">Стили</a>

Основные стили страницы

Пишем общие для всей страницы стили:


        html, body {
          margin: 0;
          max-width: 100%;
          height: 100%;
          overflow: hidden;
        }

Принцип каскада

Основным принципом в CSS является принцип каскада.

В CSS каждому правилу назначается приоритет. При обработке нескольких одинаковых правил конечным является правило с наибольшим приоритетом.

Приоритет стилей

Правила сортировки:

  1. Определяется приоритет таблицы стилей
    1. Стили браузера по умолчанию
    2. Пользовательские стили
    3. Стили страницы, подключенные через <link>
    4. Стили страницы, расположенные между <style> </style>
    5. Стили тега, прописанные в атрибуте style=""
    6. Стили с пометкой !important
  2. Определяется специфичность селектора

Задача

У нас есть несколько правил для одного и того же элемента:

  • В файле main.css
    p {color:blue;}
  • В HTML-разметке
    <style>p {color:red;}</style>
  • В атрибуте тега
    <p style="color:green;"></p>

Какого цвета будет абзац?

Специфичность

Специфичность селектора принято исчислять в баллах:

  • За каждый id – 100 баллов
  • За каждый класс – 10 баллов
  • За каждый тег – 1 балл
Селектор Специфичность
#my p#id 201 балл
#my #id 200 баллов
#my p 101 балл
.wrapper .content p 21 балл
.content p 11 баллов

Блочная модель

Любой блочный элемент состоит из нескольких слоев свойств, накладывающихся друг на друга. Основой всему служит контент (текст, изображение и др.). Управлять размерами контента можно при помощи width (ширина) и height (высота).

Дальше идут внутренние отступы – padding. При помощи этого свойства создаются пустые поля от края контента до внутреннего края границ.

Следующий слой это граница (border).

И в заключении внешние отступы блока – margin. Это свойство отвечает за пустое пространство вокруг элемента.

Ни один из этих слоев не является обязательным, даже контент.

Flexbox

Главная задумка flexbox (англ. «гибкий блок») в наделении контейнера способностью изменять ширину/высоту (и порядок) своих элементов для наилучшего заполнения пространства (в большинстве случаев — для поддержки всех видов дисплеев и размеров экранов). Flex-контейнер растягивает элементы для заполнения свободного места или сжимает их, чтобы предотвратить выход за границы. Это более эффективный способ вёрстки, выравнивания и распределения свободного места между элементами в контейнере, даже когда их размер неизвестен и/или динамический (отсюда слово «гибкий»).

Единицы измерения vw/vh

С помощью vw/vh можно устанавливать размер элементов относительно размера окна брузера. Единицы vw/vh интересны тем, что 1vw — единица равная 1/100'ой ширины окна браузера. Что бы присвоить элементу ширину, равную ширине окна, например, надо установить width:100vw.
vh в свою очередь зависит от высоты окна.

Магия

:hover

Псевдокласс :hover определяет стиль элемента когда на него наведен курсор. Может быть задан любому элементу.

Шорткат flex

Шорткатом называют короткое свойство, объединяющее несколько значений.

flex: <'flex-grow'> <'flex-shrink'> <'flex-basis'>

В нашем случае мы будем изменять значение свойства flex-grow.

Увеличиваем секцию

По наведению курсора мыши значение свойства flex-grow будет изменяться с 1 на 1.8.


        section:hover {
          flex: 1.8;
          transition: flex 0.4s;
        }

Свойство transition указывает, что изменение будет происходить плавно в течении 4 десятых секунды.

Увеличиваем текст

Размер текста также должен меняться по наведению мыши на секцию.


        section:hover > .cont_title h2 {
          font-size: 5vw;
          transition: font-size 0.4s;
        }

        section:hover > .cont_title h3 {
          font-size: 1.5vw;
          transition: all 0.4s;
        }

Прячем описание

Чтобы описание не перекрывало красивую картинку спрячем его. И будем показывать только при наведении курсора на секцию. Добавим в изначальные стили для селектора .cont_desc свойство opacity со значением 0 – это сделает текст невидимым. И напишем правила, которые будут делать текст видимым при наведении курсора.


        .cont_desc {
          opacity: 0;
        }

        section:hover > .cont_desc {
          opacity: 1;
          transition-delay: 0.5s;
          transition: opacity 0.8s;
        }

Цветной фон для описания

Зададим цветной фон для нижнего текста чтобы он хорошо читался на фоне картинок.


        .cont_desc {
          background-color: rgba(255,217,15, 0.8);
        }

Затемняем неактивные секции

Все плашки кроме той, над которой находится курсор, будут затемняться. Для этого используем свойство filter со значением brightness(x%).


        main:hover > section {
          filter: brightness(22%);
        }

        main:hover > section:hover {
          filter: brightness(100%);
        }

        section {
          transition: all 0.4s;
        }

filter

Свойство filter применяет художественный эффект или их сочетание к элементу. К фильтрам относится изменение прозрачности, добавление тени, трансформация и др.
