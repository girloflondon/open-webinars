Почему это важно

Причины писать чистый код

  • Программист больше читает кода чем пишет.
  • Много читает чужего кода, даже если работает над проектом один.
  • Код над которым вы давно не работали можно приравнять к чужому. Его писал другой человек — вы из прошлого.
  • Проекты имеют свойство развиваться.
  • Скорость чтения и понимания прямо влияет на скорость развития проекта.

Форматирование кода

Регистр символов в названиях

Шашлычный регистр в JavaScript использовать нельзя:

        
          let user-name = 'Иван';

Змеиный регистр некрасиво:

        
          let user_name = 'Иван';

Верблюжий регистр не принято:

        
          let UserName = 'Иван';

Нижний верблюжий регистр — выбор большинства:

        
          let userName = 'Иван';

Названия констант

Нужно выделять в общем коде, поэтому общий подход не годится:

        
          const daysInWeek = 7;
          const daysInMonth = 30;
          let period = 73;
          let weeks = Math.ceil(period / daysInWeek);
          let months = Math.ceil(period / daysInMonth);

Оформляем заглавными буквами в змеином регистре:

        
          const DAYS_IN_WEEK = 7;
          const DAYS_IN_MONTH = 30;
          let period = 73;
          let weeks = Math.ceil(period / DAYS_IN_WEEK);
          let months = Math.ceil(period / DAYS_IN_MONTH);

Имена классов и конструкторов

Используются с оператором new, и нет смысла давать им глагольные названия:

        
          class createShape {}
          function createUser() {}
          let circle = new createShape();
          let user = new createUser();

Называем их существительным и название даём в верхнем верблюжем регистре, для совместимости с встроенными классами:

        
          class Shape {}
          function User() {}
          let circle = new Shape();
          let user = new User();

Пробел между названием функции и скобками

При создании и при вызове функции пробел после названия «разрывает» связь с аргументами:

        
          function getNextSunday (date) {}
          let current = getNextSunday (new Date);

По соглашению функции пишем без пробела перед аргументами:

        
          function getNextSunday(date) {}
          let current = getNextSunday(new Date);

Анонимные функции

Просто нет имени, но пробел после function не часть имени:

        
          const isOdd = function(number) {
            return number % 2 === 1;
          };

Пробел между function и «именем» никуда не должен пропадать:

        
          const isOdd = function (number) {
            return number % 2 === 1;
          };

Пробел между оператором и скобками

Если не ставить пробел после оператора if или for, то их можно перепутать с вызовом функции:

        
          if(!isSunday(current)) {
            current = getNextSunday(current);
          }

Поэтому рекомендую ставить пробел:

        
          if (!isSunday(current)) {
            current = getNextSunday(current);
          }

Блоки кода

Расстановка фигурных скобок — вопрос религии. Это php-конфессия:

        
          if (!isSunday(current))
          {
            current = getNextSunday(current);
          }

Так пишут атеисты:

        
          if (!isSunday(current)) {
            current = getNextSunday(current);}

А вот так принято в JavaScript:

        
          if (!isSunday(current)) {
            current = getNextSunday(current);
          }

Точки с запятой

Можно не ставить точки с запятой и осваивать особенности ASI:

        
          function sum(a, b) {
            return
            a + b
          }

А можно расставлять точки и всегда иметь гарантированный результат:

        
          function sum(a, b) {
            return a + b;
          }

А еще записывать выражение в одну строку.

Ставить ;

        
          var isOdd = function () {}
          var user = { name: 'Иван', age: 92 }

После } при создании функционального выражения или объекта:

        
          var isOdd = function () {};
          var user = { name: 'Иван', age: 92 };

Не ставить ;

        
          function sum(a, b) {};
          if (!isSunday(current)) {
            current = getNextSunday(current);
          };

После } при создании функции или блока кода в if, while, for:

        
          function sum(a, b) {}
          if (!isSunday(current)) {
            current = getNextSunday(current);
          }

Комментарии

Складывается ощущение что это код побывал на Facebook:

        
          function hashIt(data) {
            // The hash
            let hash = 0;
            // Length of string
            const length = data.length;
            // Loop through every character in data
            for (let i = 0; i < length; i++) {
              // Get character code.
              const char = data.charCodeAt(i);
              // Make the hash
              hash = ((hash << 5) - hash) + char;
              // Convert to 32-bit integer
              hash &= hash;
            }
          }

Комментируйте только важную часть

С учетом решаемой задачи, а не просто что делает эта строка кода:

        
          function hashIt(data) {
            let hash = 0;
            const length = data.length;

            for (let i = 0; i < length; i++) {
              const char = data.charCodeAt(i);
              hash = ((hash << 5) - hash) + char;

              // Преобразуем в 32-битное целое число
              hash &= hash;
            }
          }

Убирайте закомментированный код

        
          doStuff();
          // doOtherStuff();
          // doSomeMoreStuff();
          // doSoMuchStuff();

Для хранения истории проекта или отдельных версий реализации используйте систему контроля версий кода git:

        
          doStuff();

Туда же отправляйте заметки к сделанным изменениям.

Переменные

Давайте переменным описательные названия

Что хранится в переменной x? Можно будет понять только из контекста и то не всегда:

        
          const x = new Date;

Уже лучше. В date дата, но какая? Иногда это может быть важно если разных переменных с датами у нас много:

        
          const date = new Date;

Текущая дата. Где-бы мы не встретили переменную с таким именем, сложностей никаких мы испытывать не будем:

        
          const currentDate = new Date;

Транслит в именах

Похоже на объявление станций метро в Москве на английском языке «Некст стейшн Международная»:

        
          const polzovatel = { name: 'Иван' };
          const massivChisel = [ 1, 2, 3 ];
          const cenaTovara = 5760.50;

Все операторы в JavaScript это реальные слова на английском языке, поэтому и переменные нужно называть на английском языке:

        
          const user = { name: 'Иван' };
          const numbersList = [ 1, 2, 3 ];
          const itemPrice = 5760.50;

Избегайте магических констант

Что за число 86400000? Почему такой таймаут? Загадкам не место в нашем коде:

        
          setTimeout(blastOff, 86400000);

Теперь код сам рассказывает о себе все подробности, а не играет в загадки:

        
          const MILLISECONDS_IN_A_DAY = 86400000;

          setTimeout(blastOff, MILLISECONDS_IN_A_DAY);

Используйте переменные чтобы сделать код понятнее

Что мы передаеём в функцию saveCityZipCode первым и вторым аргументом? Зато обошлись без «лишних» переменных:

        
          const address = '117105, Москва, Варшавское шоссе, 1-17';
          saveCityZipCode(address.split(', ')[1], address.split(', ')[0]);

Город, индекс, все на своих местах. Уверен, интерпретатор решит как оптимизировать этот код лучше меня:

        
          const address = '117105, Москва, Варшавское шоссе, 1-17';
          const [zipCode, city] = address.split(', ');
          saveCityZipCode(city, zipCode);

Избегайте сокращений

В строке 2 мы конечно легко понимаем что в переменной n. Но к 9 строке это понимание может улетучиться:

        
          const names = ['Иван', 'Мария', 'Ульяна'];
          names.forEach(n => {
            doStuff();
            doSomeOtherStuff();
            // ...
            // ...
            // ...
            // ...
            dispatch(n);
          });

Избегайте сокращений

        
          const names = ['Иван', 'Мария', 'Ульяна'];
          names.forEach(name => {
            doStuff();
            doSomeOtherStuff();
            // ...
            // ...
            // ...
            // ...
            dispatch(name);
          });

Не пишите «масло маслянное»

На первый взгляд все хорошо, четко и подробно. Но слово item в каждой строке, а местами и по два раза:

        
          const item = {
            itemType: 'Масло',
            itemTitle: 'Сливочное',
            itemAmount: 7,
            itemReserve: 0
          };

          function reserveItem(item, amount) {
            item.itemAmount -= amount;
            item.itemReserve += amount;
          }

Не пишите «масло маслянное»

Избавимся от излишней детализации, контекст все равно сохраняется и позволяет нам читать код без пояснений и не испытывать сложностей:

        
          const item = {
            type: 'Масло',
            title: 'Сливочное',
            amount: 7,
            reserve: 0
          };

          function reserveItem(item, amount) {
            item.amount -= amount;
            item.reserve += amount;
          }

Функции

Название функции должно говорить что она делает

Что делаете функция getNext? Очевидно что она сдвигает дату на один месяц вперед? Нет:

        
          function getNext(date) {}

          const date = getNext(new Date);

А теперь?

        
          function getNextMonth(date) {}

          const date = getNextMonth(new Date);

Связывайте название функции и аргументы

Мне кажется название функции вводит нас в заблуждение:

        
          function saveCityZipCode(zipCode, city) {}

          saveCityZipCode(city, zipCode);

Нужно менять порядок либо в названии, либо в аргументах, чтобы название помогало нам:

        
          function saveCityZipCode(city, zipCode) {}

          saveCityZipCode(city, zipCode);

Значения по умолчанию вместо проверок и сокращенных вычислений

        
          function createMicrobrewery(title) {
            const breweryName = title || 'Хипстерская Пивная компания';
            // ...
          }

Как минимум значение по умолчанию будет видно без анализа тела функции:

        
          function createMicrobrewery(breweryName = 'Хипстерская Пивная компания') {
            // ...
          }

Два аргумента хорошо, три не очень, четыре — ужасно

Что же делать с функцией, требующей много информации из вызывающего кода, или которая поддаётся тонкой настройке:

        
          function createMenuItem(title, action, shortName, hotKey, confirmable) {
            // ...
          }

Часть необязательных аргументов, а возможно и все, можно принять в виде объекта:

        
          function createMenuItem(title, action, { shortName, hotKey, confirmable }) {
            // ...
          }

          createMenuItem('Сохранить изменения', action, {
            shortName: 'Сохранить',
            hotKey: 'Ctrl+S',
            confirmable: false
          })

Одна задача — одна функция

Если функция делает больше чем одну задачу, она становится менее универсальной. Эта функция считает площадь и выводит её:

        
          function getBoxSurfaceArea(x, y, z) {
            const area = 2 * (x * y + x * z + y * z);
            console.log(`Пладь поверхности: ${area}`);
            return area;
          }
          getBoxSurfaceArea(1, 2, 3);

Оставим только одно:

        
          function getBoxSurfaceArea(x, y, z) {
            return 2 * (x * y + x * z + y * z);
          }
          const area = getBoxSurfaceArea(1, 2, 3);
          console.log(`Пладь поверхности: ${area}`);

Выносите низкоуровневые детали в отдельные функции

Что нам мешает понять что и как делает функция extractRandomItem?

        
          function extractRandomItem(list) {
            const index = Math.floor(Math.random() * list.length);
            return list.splice(index, i);
          }

          const names = ['Иван', 'Олег', 'Илья'];
          const name = extractRandomItem(names);

Непонятные манипуляции во второй строке!

Выносите низкоуровневые детали в отдельные функции

Теперь любой легко поймет что делает функция extractRandomItem и даже понять как она это делает:

        
          function getRandomInt(from, to) {
            const diff = from - to + 1;
            return Math.floor(Math.random() * diff) + from;
          }
          function extractRandomItem(list) {
            const index = getRandomInt(0, list.length - 1);
            return list.splice(index, i);
          }

          const names = ['Иван', 'Олег', 'Илья'];
          const name = extractRandomItem(names);

Сложные условия заслуживают отдельной функции

        
          if (fsm.state === 'fetching' && isEmpty(listNode)) {
            // ...
          }

Условия имеют свойства рости в процессе, теперь нас это не беспокоит:

        
          function shouldShowSpinner(fsm, listNode) {
            return fsm.state === 'fetching' && isEmpty(listNode);
          }

          if (shouldShowSpinner(fsm, listNode)) {
            // ...
          }

Флаги в аргументах указывают на многозадачность функции

        
          function createFile(name, temp) {
            if (temp) {
              fs.create(`./temp/${name}`);
            } else {
              fs.create(name);
            }
          }

Флаги в аргументах указывают на многозадачность функции

        
          function createFile(name) {
            fs.create(name);
          }

          function createTempFile(name) {
            createFile(`./temp/${name}`);
          }

Избегайте побочных эффектов

Если на то нет причин: возвращайте результат, а не присваивайте его в переменную замыкания:

        
          let name = 'Олег Петров';

          function splitName() {
            name = name.split(' ');
          }

          splitName();

          console.log(name);
          // [Олег, Петров];

Избегайте побочных эффектов

Тогда функцию splitIntoFirstAndLastName можно будет повторно использовать без опасений:

        
          let name = 'Олег Петров';

          function splitName(name) {
            return name.split(' ');
          }

          const [firstName, lastName] = splitName(name);

          console.log(firstName); // Олег
          console.log(lastName); // Петров

Используйте возможности массивов

        
          const programmerOutput = [
            { name: 'Uncle Bobby', linesOfCode: 500 },
            { name: 'Suzie Q', linesOfCode: 1500 },
            { name: 'Jimmy Gosling', linesOfCode: 150 },
            { name: 'Gracie Hopper', linesOfCode: 1000 }
          ];

          let totalOutput = 0;

          for (let i = 0; i < programmerOutput.length; i++) {
            totalOutput += programmerOutput[i].linesOfCode;
          }

Используйте возможности массивов

        
          const programmerOutput = [
            { name: 'Uncle Bobby', linesOfCode: 500 },
            { name: 'Suzie Q', linesOfCode: 1500 },
            { name: 'Jimmy Gosling', linesOfCode: 150 },
            { name: 'Gracie Hopper', linesOfCode: 1000 }
          ];

          const INITIAL_VALUE = 0;

          const totalOutput = programmerOutput
            .map(programmer => programmer.linesOfCode)
            .reduce((total, linesOfCode) => {
              return total + linesOfCode;
            }, INITIAL_VALUE);
