# Пишем чистый JavaScript код

Для кого программист пишет качественный код? Его что будет исполнять интерпретатор-педант? Мы можем задеть его чувственные машинные коды?

Разберем эти и другие вопросы написания качественного кода на языке JavaScript. Выясним что значит качественный код. А главное, разберем огромное кол-во примеров как плохой код сделать легко читаемым, удобным для повторного использования, и простым в улучшении. Попутно выясним что такое плохой код, и чем же он плох.

Конечно знание как писать качественный код, не сделает вас сразу хорошим программистом. Но оно поможет вам анализировать ваш код, искать проблемные места и улучшать их. И чем чаще вы будете это делать, тем быстрее и проще первый черновик будет превращаться в элегантный фрагмент кода, который хоть сейчас можно публиковать в учебнике по программированию.

Важно ли уже уметь программировать, чтобы задуматься о качестве кода? Нет. Чем раньше вы научитесь оценивать результат, тем меньше нужно будет переучиваться потом.
